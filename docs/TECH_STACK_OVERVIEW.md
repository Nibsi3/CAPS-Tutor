## PDF Extraction Pipeline

### Stage 1: Low-Level PDF Extraction

**Technology Stack:**
- **Python 3.x** with **PyMuPDF (fitz)** library
- **OpenCV (cv2)** for image processing and diagram detection
- **NumPy** for image manipulation
- **Ollama LLM** (local) or **Groq API** for semantic understanding

**Location:** `C:\Users\cameron\Documents\Poppler\pdf_parser_pipeline.py`

**Process:**

1. **PDF Text Extraction**
   ```python
   # Uses PyMuPDF to extract:
   - Text blocks with bounding boxes (x0, y0, x1, y1)
   - Images with coordinates
   - Vector shapes
   - Page metadata
   ```

2. **Diagram Detection**
   - Uses OpenCV to detect visual content
   - Filters by area ratio (2%-85% of page)
   - Filters by aspect ratio (0.2-5.0)
   - Detects visual variance (minimum 100 variance threshold)
   - Identifies diagrams vs. text blocks

3. **Diagram Identification with LLM**
   - Links diagrams to question numbers using spatial analysis
   - Uses LLM to classify diagram types (diagram, graph, table)
   - Extracts diagram labels and titles
   - Matches diagrams to questions based on Y-position (questions usually appear after diagrams)

        }
      ],
      "diagrams": [
        {
          "bbox": [x, y, w, h],

### Stage 2: Semantic Interpretation

**Process:**
- Parses exam structure (sections, questions, subquestions)
- Detects question types using pattern matching and LLM
- Extracts marks, options, and metadata
- Uses LLM for complex semantic understanding

**Question Type Detection:**
- **Multiple Choice:** Detects A, B, C, D options
- **Diagram:** Questions referencing diagrams/figures
- **Free Text:** Standard text questions
- **True/False:** Boolean questions
- **Matching:** Column matching questions

### Stage 3: Normalization

**Process:**
- Converts structured data into predictable fields
- Links diagrams to questions
- Validates and sorts questions
- Prepares data for editor consumption

**Output:** Normalized JSON structure ready for upload

## Question Ordering System

### Order Calculation Algorithm

**Location:** `src/lib/question-storage.ts`

**Function:** `calculateQuestionOrder(questionNumber: string): number`

**Algorithm:**
```typescript
// Converts question numbers to sortable numeric values
// Example: "1.1" -> 1.1, "2.3.1" -> 2.31

function calculateQuestionOrder(questionNumber: string): number {
  const parts = questionNumber.split('.').map(Number);
  let order = 0;
  let multiplier = 1;
  
  // Reverse to process from right to left
  for (let i = parts.length - 1; i >= 0; i--) {
    order += parts[i] * multiplier;
    multiplier *= 100; // Allow up to 99 sub-questions per level
  }
  
  return order;
}
```

**Examples:**
- `"1.1"` → `1.1` → stored as `1100` (multiplied by 1000)
- `"1.2"` → `1.2` → stored as `1200`
- `"2.1"` → `2.1` → stored as `2100`
- `"1.1.1"` → `1.11` → stored as `1110`
- `"1.1.2"` → `1.12` → stored as `1120`
- `"2.3.1"` → `2.31` → stored as `2310`

**Storage:**
- Order value stored as integer: `Math.round(order * 1000)`
- Preserves 3 decimal places as integer
- Allows proper sorting in database queries

### Alternative Order Calculation (Poppler JSON)

**Location:** `src/app/api/admin/past-papers-v2/upload-poppler-json/route.ts`

**Function:** `calculateOrder(questionNumber: string): number`

**Algorithm:**
```typescript
// Format: Q1.1 -> 1001000, Q1.1.1 -> 1001001, Q1.2 -> 1002000
function calculateOrder(questionNumber: string): number {
  const parts = questionNumber.split('.');
  let order = 0;
  for (let i = 0; i < parts.length; i++) {
    const part = parseInt(parts[i]) || 0;
    order += part * Math.pow(1000, 2 - i);
  }
  return order;
}
```

## Question Types

### Supported Question Types

1. **`multiple-choice`** (MCQ)
   - Has options array: `["A) Option 1", "B) Option 2", ...]`
   - Rendered as radio buttons
   - Single correct answer

2. **`free-text`** (Standard)
   - Open-ended text response
   - Rendered as textarea
   - Used for short and long answer questions

3. **`diagram`**
   - Has associated image (`imageFileId`)
   - Image displayed above or below question
   - Used for questions referencing diagrams/figures

4. **`true-false`**
   - Boolean question
   - Rendered as True/False radio buttons
   - Can be auto-detected from question text

5. **`graph`** (Specialized diagram)
   - Treated as diagram type
   - For graph/chart questions

6. **`table`** (Specialized diagram)
   - Treated as diagram type
   - For table-based questions

### Type Detection Logic

**During Upload:**
```typescript
// Check for multiple choice
if (questionText.match(/^[A-D]\)/m) || questionText.includes('A)') && questionText.includes('B)')) {
  type = 'multiple-choice';
}

// Check for diagram
if (questionText.match(/diagram|figure|graph|table|image|refer to|study the/i)) {
  type = 'diagram';
}

// Check for true/false
if (questionText.match(/true|false|correct|incorrect/i)) {
  type = 'true-false';
}

// Default
if (!type) {
  type = 'free-text';
}
```

**During Rendering:**
```typescript
// Parse MCQ options from question text if not in database
const parseMCQOptions = (text: string) => {
  const mcqPattern = /^(.+?)\n([A-D]\)\s+.+?)(?:\n([A-D]\)\s+.+?))?(?:\n([A-D]\)\s+.+?))?(?:\n([A-D]\)\s+.+?))?$/m;
  const match = text.match(mcqPattern);
  if (match) {
    return {
      questionStem: match[1],
      options: [match[2], match[3], match[4], match[5]].filter(Boolean)
    };
  }
  return null;
};
```
