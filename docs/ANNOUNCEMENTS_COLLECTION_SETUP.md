# Announcements Collection Setup Guide

## Collection Details

**Collection ID:** `announcements` (lowercase, exactly as shown)

## Required Attributes

Create the following attributes in your Appwrite `announcements` collection:

### 1. title
- **Type:** String
- **Size:** 255 characters
- **Required:** Yes
- **Array:** No
- **Description:** The title/heading of the announcement

### 2. message
- **Type:** String
- **Size:** 5000 characters (or larger if needed)
- **Required:** Yes
- **Array:** No
- **Description:** The main content/message of the announcement

### 3. priority
- **Type:** String
- **Size:** 50 characters
- **Required:** No (optional)
- **Array:** No
- **Default Value:** `medium`
- **Description:** Priority level (e.g., 'low', 'medium', 'high', 'urgent')
- **Note:** The API defaults to 'medium' if not provided

### 4. active
- **Type:** Boolean
- **Required:** No (optional)
- **Array:** No
- **Default Value:** `true`
- **Description:** Whether the announcement is currently active/visible
- **Note:** The API defaults to `true` if not provided

### 5. createdBy
- **Type:** String
- **Size:** 255 characters
- **Required:** Yes
- **Array:** No
- **Description:** The user ID of the admin who created the announcement

### 6. createdAt
- **Type:** String (or DateTime if Appwrite supports it)
- **Size:** 255 characters
- **Required:** No (optional)
- **Array:** No
- **Description:** ISO timestamp of when the announcement was created
- **Note:** Can be auto-generated by Appwrite or set by the API

### 7. targetAudience
- **Type:** String
- **Size:** 50 characters
- **Required:** No (optional)
- **Array:** No
- **Default Value:** `both`
- **Allowed Values:** `students`, `admins`, or `both`
- **Description:** Specifies who should see this announcement
  - `students`: Only visible to students (when not in admin mode)
  - `admins`: Only visible to administrators (when in admin mode)
  - `both`: Visible to everyone (default)

### 8. scheduledStart
- **Type:** String (ISO 8601 datetime format)
- **Size:** 255 characters
- **Required:** No (optional)
- **Array:** No
- **Description:** When the announcement should start being displayed
- **Note:** If not set, the announcement starts immediately when active
- **Example:** `2024-12-25T09:00:00.000Z`

### 9. scheduledEnd
- **Type:** String (ISO 8601 datetime format)
- **Size:** 255 characters
- **Required:** No (optional)
- **Array:** No
- **Description:** When the announcement should stop being displayed
- **Note:** If not set, the announcement continues until manually deactivated
- **Example:** `2024-12-31T23:59:59.000Z`

## Permissions

Set the following permissions for the `announcements` collection:

### Read Permissions
- **Users:** Read access (so all users can see active announcements)
- **Admins:** Full read access

### Create Permissions
- **Admins only:** Only administrators should be able to create announcements

### Update Permissions
- **Admins only:** Only administrators should be able to update announcements

### Delete Permissions
- **Admins only:** Only administrators should be able to delete announcements

## Indexes (Optional but Recommended)

Create indexes for better query performance:

1. **active index**
   - Attribute: `active`
   - Type: Key
   - Order: ASC

2. **createdAt index**
   - Attribute: `$createdAt` (Appwrite's built-in timestamp)
   - Type: Key
   - Order: DESC

3. **priority index** (if you want to filter by priority)
   - Attribute: `priority`
   - Type: Key
   - Order: ASC

## Step-by-Step Creation in Appwrite Console

1. **Go to Appwrite Console**
   - Navigate to your project
   - Click on "Databases" in the left sidebar
   - Select your database

2. **Create Collection**
   - Click "Create Collection"
   - Enter Collection ID: `announcements` (lowercase, exactly)
   - Click "Create"

3. **Add Attributes**
   - Click on the "Attributes" tab
   - Add each attribute listed above using "Create Attribute"
   - Make sure to set the correct type, size, and required status

4. **Set Permissions**
   - Click on the "Permissions" tab
   - Configure read/write/delete permissions as described above

5. **Create Indexes (Optional)**
   - Click on the "Indexes" tab
   - Create the recommended indexes for better performance

## Verification

After creating the collection, test it by:

1. Creating an announcement through the admin panel
2. Verifying it appears in the announcements list
3. Checking that active announcements are visible to users

## Example Document Structure

### Basic Announcement (Immediate, All Users)
```json
{
  "$id": "unique-document-id",
  "title": "System Maintenance Notice",
  "message": "The system will be under maintenance on Saturday from 2 AM to 4 AM.",
  "priority": "high",
  "active": true,
  "createdBy": "user-id-of-admin",
  "createdAt": "2024-01-15T10:30:00.000Z",
  "targetAudience": "both"
}
```

### Scheduled Announcement (Students Only)
```json
{
  "$id": "unique-document-id-2",
  "title": "Holiday Break Notice",
  "message": "The platform will be unavailable during the holiday break.",
  "priority": "medium",
  "active": true,
  "createdBy": "user-id-of-admin",
  "createdAt": "2024-12-20T10:30:00.000Z",
  "targetAudience": "students",
  "scheduledStart": "2024-12-25T00:00:00.000Z",
  "scheduledEnd": "2025-01-05T00:00:00.000Z"
}
```

### Admin-Only Announcement
```json
{
  "$id": "unique-document-id-3",
  "title": "Admin Meeting Reminder",
  "message": "Don't forget about the admin meeting tomorrow at 2 PM.",
  "priority": "low",
  "active": true,
  "createdBy": "user-id-of-admin",
  "createdAt": "2024-01-15T10:30:00.000Z",
  "targetAudience": "admins"
}
```

## Troubleshooting

If you still get errors after creating the collection:

1. **Check Collection ID:** Must be exactly `announcements` (lowercase, no spaces)
2. **Verify Attributes:** All required attributes must exist with correct types
3. **Check Permissions:** Ensure your API key has the necessary permissions
4. **Restart Server:** After creating the collection, restart your Next.js dev server
