# Upload Past Papers to Appwrite Storage

This script uploads all PDF files from the `past papers` directory to your Appwrite Storage bucket.

## Prerequisites

1. **Create an Appwrite API Key**
   - Go to [Appwrite Console](https://cloud.appwrite.io)
   - Navigate to **Settings** → **API Keys**
   - Click **Create API Key**
   - Give it a name (e.g., "Past Papers Uploader")
   - Select the following scopes:
     - ✅ `storage.read`
     - ✅ `storage.write`
   - Click **Create**
   - **Copy the API key** (you won't be able to see it again!)

2. **Add API Key to Environment Variables**
   - Open your `.env.local` file in the project root
   - Add the following line:
     ```
     APPWRITE_API_KEY=standard_cb8d1e93a8899b8392b9fcf82ea9b47fff96cdde29fae8c062726fee05d5dfa38a3825ea20e244f7c017274b2f443f3d449c361cd8ae348392ffda434b4bf746c61b17d9f94e590026aa9397cc8c6b32eaa7c19aab9c6009eb26d51c45572ece49ac9e08ab14b55dac1375192d82a9870ee26f329a9d72be3df6936c97009451
     ```
   - Replace `your_api_key_here` with the API key you just copied
   - Save the file

## Running the Script

### Option 1: Using Node.js directly
```powershell
node scripts/upload-past-papers-to-appwrite.mjs
```

### Option 2: Using npm script (if added to package.json)
```powershell
npm run upload-past-papers
```

## Configuration

The script uses these environment variables (from `.env.local`):

- `APPWRITE_ENDPOINT` - Default: `https://fra.cloud.appwrite.io/v1`
- `APPWRITE_PROJECT_ID` or `NEXT_PUBLIC_APPWRITE_PROJECT_ID` - Your Appwrite project ID
- `APPWRITE_API_KEY` - **Required** - Your API key with storage permissions
- `APPWRITE_BUCKET_ID` - Default: `690dafea0021f232399e` (the past paper bucket)

## What the Script Does

1. ✅ Scans the `past papers` directory recursively for all files
2. ✅ Verifies the Appwrite bucket exists and is accessible
3. ✅ Uploads files in batches (5 at a time) to avoid rate limits
4. ✅ Skips files that already exist
5. ✅ Shows progress and a summary at the end

## Output

The script will display:
- ✅ Files successfully uploaded
- ⏭️ Files skipped (already exist)
- ❌ Files that failed to upload
- 📊 Final summary with counts

## Troubleshooting

### "API_KEY is not set"
- Make sure you've added `APPWRITE_API_KEY` to your `.env.local` file
- Restart your terminal after adding the variable

### "Bucket not found"
- Verify the bucket ID `690dafea0021f232399e` exists in your Appwrite project
- Check that your API key has the correct permissions

### "Permission denied"
- Ensure your API key has both `storage.read` and `storage.write` scopes
- Verify the bucket permissions allow file uploads

### Upload fails for large files
- Check the bucket's maximum file size limit in Appwrite Console
- The script shows the maximum file size when it starts

## Notes

- Files are uploaded with public read access (`read("any")`) by default
- Each file gets a unique ID generated by Appwrite
- The original filename is preserved in the file metadata
- The script processes files in batches to avoid overwhelming the API
